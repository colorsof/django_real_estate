## LEARNING PHASES (6:30am-9:30am)

### Phase 1: Learn Concept (30-45 min)
1. Watch course lecture (don't code yet)
2. Ask Claude: ELI5 ‚Üí ELI10 ‚Üí Developer explanation
3. Understand WHY before HOW

### Phase 2: Understand Approach (15-20 min)
Ask Claude: "How would YOU build this?"
Compare to course approach later

### Phase 3: Watch Course (10-15 min)
See how instructor does it, note differences

### Phase 4: Compare (10-15 min)
Ask Claude to explain differences educationally:
- Why course chose their way
- Why Claude suggested different
- Trade-offs for LEARNER

### Phase 5: Code (60-90 min)
Type code, ask "why" constantly:
- "Why are we doing X?"
- "Walk me through the flow"
- "What did I misunderstand?"

### Phase 6: Test & Debug (20-30 min)
Make it work, understand WHY it works

### Phase 7: Commit & Reflect (10 min)
```bash
git commit -m "feat: [Feature] - [What I learned]
- Key learning: [concept mastered]
- Used Claude for [challenge]
- Lecture: #[X]"
```

Write session notes with timestamps of key moments

### 9:30am - STOP RECORDING
Save, take break. Video exists, just needs editing.

## THE PROMPTING FRAMEWORK

### Level 1: Understanding
```
"Explain [CONCEPT]:
1. Like I'm 5
2. Like I'm 10
3. Like I'm a developer

Then: Why exists? What breaks without it?"
```

### Level 2: Planning
```
"My task: [feature]
How would YOU approach this?
What are key decisions?
What's simplest for learner?"
```

### Level 3: Compare Approaches
```
"You suggested [X], course does [Y]
Explain educationally:
- Why each approach?
- Trade-offs?
- Which for LEARNER?"
```

### Level 4: Implementation
```
"Let's build [feature]
Explain what each piece does and WHY
Point out deep understanding vs boilerplate"
```

### Level 5: Debugging
```
"Error: [paste]
Code: [paste]
What did I misunderstand?
Don't just fix - help me understand"
```

### Level 6: Test Understanding
```
"Let me explain [concept] back to you:
[Your explanation]
Is this correct? What am I missing?"
```
